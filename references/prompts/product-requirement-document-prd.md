# Product Requirement Document (PRD)

- Use when: a one-pager must capture context, problem, solution, scope, risks, dependencies, and metrics for decision-making
- Phase: Product
- Type: Prompt
- Rating: 5/5

## Prompt Structure

- Role: Senior Product Manager
- Objective: craft a decision-ready PRD one-pager
- Required sections:
  - Context and background
  - User problem
  - Business problem
  - Hypothesis
  - Success metrics
  - Proposed solution
  - Out of scope
  - Alternatives considered
  - MVP scope
  - Key risks and mitigations
  - Dependencies
  - Open questions

## Output Rules

- Write in markdown with clear headings.
- Use concise, business-ready language.
- Keep sections short, scannable, and decision-oriented.
- Include a plain-language solution description and simple user flow.
