# Problem Framing

- Use when: the team needs a clear problem statement grounded in evidence and reframed into opportunities
- Phase: Product
- Type: Prompt
- Rating: 5/5

## Required Sections

- Problem Statement
- Evidence
- Impact
- Users and Context
- How Might We
- Desired Outcomes
- Success Metrics

## Output Rules

- Use bullet points.
- Keep the problem statement short and user-focused.
- Evidence should cite facts, observations, or anecdotes.
- Success metrics should be measurable and outcome-oriented.
