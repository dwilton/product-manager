# Product Requirements Document (PRD) Template

Use this as the default PRD structure when the user asks for a one-pager or concise PRD.

## Sections

- Author / Stakeholders
- Context and Goal
  - Background
  - Goals
- Problem
  - User Problem
  - Business Problem
- Hypothesis
- Assumptions
- Success Metrics
- Proposed Solution
  - User Interaction Design
  - Out of Scope
  - Alternatives Considered
- MVP Scope
- Challenges and Mitigation
- Dependencies
- Open Questions
- Appendix / References

## Guidance

- Start concise and decision-ready.
- Keep deeper research, designs, or appendices linked rather than embedded.
- Evolve the one-pager into a fuller PRD only when complexity or risk increases.
