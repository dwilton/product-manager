# Release Notes



**Release Notes** communicate new features, improvements, and fixes to customers in a clear, consistent way that highlights customer value.

Use this template for every release to maintain consistency and ensure customers understand what changed and why it matters.

Keep notes short and benefit-focused — link to documentation for details.

## **Template Structure**

**[Date] Release Notes – [Company/Product Name]**

**What's New**

- **Feature Name** – How this helps customers

**Improvements**

- **Enhancement** – What changed and customer benefit

**Fixes**

- **Bug fix** – What was resolved

**Rollout Notes** (if needed)

- Feature status and customer actions required

**Resources**

- Links to documentation and support

## **Example**

**15 Aug 2025 Release Notes – Product X**

**What's New**

- **Advanced Search Filters** – Find exactly what you need with multiple filter options
- **Custom Dashboards (Beta)** – Create personalized views of your data (contact support to join beta)

**Improvements**

- Faster page loading across the application
- Clearer mobile navigation for easier workflows

**Fixes**

- Resolved file upload failures
- Fixed incorrect time zone display in notifications

**Resources**

- [Help Article: Using Advanced Filters]
- [Contact Support]

## **Writing Guidelines**

- Write for customers, not internal teams
- Lead with customer value, not technical details
- Keep descriptions short and benefit-focused
- Lead with positive new features, end with fixes
- Include links to help customers use new features
- Maintain a helpful and accessible tone
