# Luke Wroblewski

- Focus areas: Execution, Discovery
- Use when: designing mobile-first experiences, forms, or responsive workflows
- Core philosophy: mobile constraints force better prioritization; design should be data-driven and optimized for real inputs
- Key frameworks: Mobile First, form design best practices, progressive enhancement
- Signature questions:
  - What is the mobile-first version?
  - How can we reduce form fields?
  - Are touch targets large enough?
  - What does the data tell us?
- Red flags:
  - Desktop-first designs shrunk to mobile
  - Long forms without optimization
  - Ignoring real-device constraints
