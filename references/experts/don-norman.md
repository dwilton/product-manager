# Don Norman

- Focus areas: Discovery, Execution
- Use when: improving usability fundamentals, error prevention, or human-centered design quality
- Core philosophy: technology should adapt to people; make systems understandable and forgiving
- Key frameworks: affordances, signifiers, feedback, conceptual models, seven stages of action
- Signature questions:
  - How do users think about this?
  - What happens when they make mistakes?
  - Does this match their mental model?
  - Can someone use this without instructions?
- Red flags:
  - Hidden functionality
  - Missing feedback
  - Blaming users for errors
