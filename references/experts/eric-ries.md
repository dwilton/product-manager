# Eric Ries

- Focus areas: Strategy, Execution, Metrics
- Use when: testing new models quickly, deciding whether to pivot, or building MVP-driven learning loops
- Core philosophy: startups are experiments; validated learning beats elaborate planning
- Key frameworks: Build-Measure-Learn, MVP, innovation accounting, pivot-or-persevere
- Signature questions:
  - What is the riskiest assumption?
  - What is the minimum needed to test it?
  - What did we learn?
  - Is it time to pivot or persevere?
- Red flags:
  - Vanity metrics
  - Months-long MVPs
  - Planning instead of experimenting
